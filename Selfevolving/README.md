# Selfevolving

Task notes for execution improvement.

---

## Use

Keep one markdown file per task in this folder. Use it to track the best first move, tool ratings, problems, and reusable corrections.

---

## Name

Use a stable, short filename. Prefer `YYYY-MM-DD-short-task.md` or `short-task.md` when the task stays active.

---

## Update

Reuse the same file while the task is active. Start a new file only when the task changes materially.

---

## Record

Each note should capture what was done, what failed, what should have happened first, and what improves speed or correctness next time. Promote durable lessons to persistent memory after the task.

---

## Avoid

Do not store secrets, credentials, or unrelated project notes here. Do not create multiple files for the same task unless the work split is real.

---

## Example

- `2026-04-07-skill-docs.md`
- `graph-index-debug.md`
- `sdk-regen-path.md`
